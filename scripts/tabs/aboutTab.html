这个在线工具的目的是可视化被http抓包工具产生的 HTTP Archive (HAR) 文. 这些文件包含HTTP日志 客户端/服务器会话，可用于对页面加载性能进行分析.

此工具的用户界面由以下选项卡组成：

  • Home - 将日志文件的内容粘贴到此选项卡的文本框中.
  • Preview - 如果要查看可视化的HTTP流量，请切换到此选项卡.
  • HAR - 使用此选项卡可以将输入JSON数据的结构视为可扩展树.
  • Schema - 在此选项卡中浏览输入日志的格式.

使用Firebug创建HARs

可以用来记录通过HTTP协议传输的数据的工具之一 是 Firebug. 为此，您还必须安装已调用扩展名的Firebug NetExport.

HTTP Archive 规范

使用JSON模式描述了输入HTTP存档文件（.har）所需的结构。您可以在本页的“架构”选项卡中浏览当前架构定义。

请求计时字段

HTTP日志的一部分也是关于网络请求执行的计时信息。 以下是对各个请求/响应阶段的描述：

  • Blocking - 排队等待网络连接所用的时间.
  • DNS Lookup - DNS解析时间。解析主机名所需的时间.
  • Connecting - 创建TCP连接所需的时间.
  • Sending - 向服务器发送HTTP请求所需的时间.
  • Waiting - 等待服务器的响应.
  • Receiving - 从服务器（或缓存）读取整个响应所需的时间.

在线日志文件

HAR查看器还支持JSONP，因此可以加载日志文件 来自不同的领域。这允许链接在线日志并预览它们 在查看器中自动执行。 查看在线 例子.

1. *.har文件的内容必须包含在回调函数中:

onInputData({ "log": { ... } })

2. 显示*.har文件的链接（使用此查看器）必须指定 文件输入 inputUrl:

http://www.softwareishard.com/har/viewer/?inputUrl=http://www.example.com/netData.har

3.回调函数的自定义名称可以在一个 callback参数中指定 （默认情况下为 onInputData ）：

http://www.softwareishard.com/har/viewer/?inputUrl=http://www.example.com/netData.har&callback=onInputData
